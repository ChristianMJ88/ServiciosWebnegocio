Configuración de WhatsApp por tenant

Estado general
Recomendación SaaS: usa "Subcuenta Twilio" para cada empresa. Así separas sender, plantillas y operación por tenant sin mezclar todo en una sola cuenta operativa.
Habilitar notificaciones y conversación por WhatsApp para este tenant Account SID Auth Token nuevo (opcional) Modelo de cuenta Twilio Cuenta plataforma Subcuenta Twilio Cuenta propia del tenant Subaccount SID (opcional)
Provisionamiento guiado
Friendly name de la subcuenta
Este botón crea la subcuenta en Twilio usando tu cuenta plataforma y guarda el `Subaccount SID` cifrando el `Auth Token` del tenant. Después aún debes registrar el sender de WhatsApp, el Messaging Service y las plantillas del negocio.
Friendly name del Messaging Service Webhook entrante del Messaging Service (opcional)
Este botón crea el `Messaging Service` dentro de la cuenta efectiva del tenant. El siguiente paso es capturar el `Channel Sender SID` (`XE...`) del sender de WhatsApp y asociarlo al servicio.
Channel Sender SID (XE...)
Puedes obtener este `XE...` desde Twilio Console o Senders API. Este paso conecta el sender de WhatsApp del negocio con el `Messaging Service` del tenant.

Número remitente Messaging Service SID Channel Sender SID Status callback URL
Meta y sender
Business display name Sender phone number Sender status Quality rating Throughput (MPS) WABA ID Meta Business Manager ID
Plantillas activas
SID plantilla solicitud de confirmación SID plantilla cita reprogramada pendiente SID plantilla recordar confirmar cita SID plantilla cita confirmada SID plantilla recordatorio SID plantilla cancelación por negocio SID plantilla liberada sin confirmación SID plantilla gracias por tu visita SID plantilla recordatorio de regreso
Si dejas vacío el Auth Token, el backend puede conservar el secreto actual. Usa "Cuenta plataforma" si operas todo bajo tu Twilio, "Subcuenta" si crearás una por empresa y "Cuenta propia del tenant" si el cliente traerá sus propias credenciales. La plantilla de solicitud de confirmación se usa al registrar la cita y debe pedir al cliente que responda para confirmar su asistencia. La de cita reprogramada pendiente se usa cuando la cita cambia de fecha y vuelve a estado pendiente. La de recordar confirmar se usa cuando la cita sigue pendiente y se acerca la fecha. La de cancelación se reserva para mensajes iniciados por el negocio. Las nuevas de liberada, agradecimiento y regreso quedan listas para la siguiente fase de automatización.
